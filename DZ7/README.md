# Домашнее задание: DZ7 - Авторы и текст из Файла 

## Описание проекта

Проект реализован на основе примера **Razor_Result**.  
В зависимости от параметров запроса `?name=` и `mode=` сервер возвращает данные в разных форматах:

- обычный текст (Content)
- файл для скачивания
- JSON
- перенаправление (Redirect) на сайт Яндекс, если файл не найден

---

## Структура данных

Для каждого автора используется отдельный текстовый файл в папке `Data`:

Data/
├── Пушкин.txt
├── Шекспир.txt

Содержимое файла — текст отрывка из произведения.

---

## Примеры запросов

### 1. Обычный вывод текста (Content)
Содержимое файла — текст произведения или отрывок.

https://localhost:7002/?name=Пушкин
https://localhost:7002/?name=Шекспир

Выводит содержимое соответствующего файла на странице.

---

### 2. Скачать файл

https://localhost:7002/?name=Пушкин&amp;mode=file

Файл отправляется пользователю как вложение для скачивания.

---

### 3. Получить данные в формате JSON

https://localhost:7002/?name=Пушкин&amp;mode=json

Ответ возвращается в формате:

```json
{
  "Author": "Пушкин",
  "Text": "..."
}

#### ВАЖНО: Особенность JSON

JSON — это формат передачи данных между программами.
Программы, которые читают JSON, понимают, что символ \n означает новую строку.

Однако браузер отображает JSON как «сырой» текст,
поэтому переносы строк видны как спецсимволы \n, а не как обычный текст.

В коде была предпринята попытка улучшить читаемость JSON:

заменены \r\n на \n

включён форматированный вывод (WriteIndented = true)

Тем не менее, браузер всё равно отображает переносы строк как спецсимволы,
что является нормальным поведением для JSON-ответа.

### 4. Redirect (если файл не найден)

Если файл не существует, выполняется перенаправление на сайт Яндекс:

https://localhost:7002/?name=Толстой

Происходит переход на:

https://yandex.ru/search/?text=Толстой
